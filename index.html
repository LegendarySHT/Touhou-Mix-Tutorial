消歧义

  • THMIX：

    指TouHou Mix

  • 官网：

    指thmix的官网(thmix.org或asia.thmix.org)

  • 专辑界面：

    指进入游戏后的让玩家选择专辑的初始界面

  • 商店界面：

    指进入游戏后，右下角的按钮(需要开启线上模式)点进去后的界面

  • Midi：

    指一种特定格式的文件，THMIX的铺面均是通过Midi生成的。有时候也用于指代游戏内的歌曲，因为本质上没有太大的区别

  • Midi信息界面：

    指选择完专辑及歌曲后进入的界面

  • 音轨界面：

    指点击Midi信息界面左下角的音符按钮进入的页面

  • 总设置：

    指屏幕顶栏上右边的设置按钮点进去后的地方

  • 游戏设置：

    指歌曲马上开始时，显示在屏幕中间的设置按钮点进去后的地方

  • Note(音符)：

    指打歌时落下的那些音符，包括点键(点块)，滑键(滑块)和长条

  • 游戏文件目录：

    指THMIX存放midi、数据和资源文件等内容使用的文件夹

  • 人声音频

    THMIX可以通过将音频文件(如MP3)与midi同步播放来弥补midi缺少人声的问题，所以人声音频只是指外部导入的音频文件，不一定只有人声

  • pp：

    指Performance Point(计算方式为 ln(1+「总分数」) × 「准确度」 2 ) ，是对成绩排名的依据，游戏账号的等级也是根据pp值计算的(计算方式为 向下取整( 「总pp值」 ) )

新手教程

如果你是第一次玩THMIX，那么我十分感谢你能来看这个教程，而不是连一些比较基本的问题都需要在群里问 此新手指引旨在让不熟悉THMIX的人也能较为快速地调出手感好的配置，所以此部分仅对关键的内容作介绍（ 基础教程必看，进阶教程可不看


  1. 基础教程
    • 总设置的配置
      • 打开Online Mode(线上模式)，除非你不需要下载歌曲之类的联网功能，更改后需要重启游戏才能生效
      • 将Display Language(游戏语言)修改为简体中文(Chinese(Simplified))或繁体中文(Chinese(Traditional))
      • 关闭全局内容翻译，除非你希望歌曲的名字变成机翻或者不希望内置歌曲名字没有翻译
      • 检查顶栏能否显示出网络延迟，或Midi商店能否正常下载歌曲。若不能，更改服务器网络节点，请先尝试全球和亚洲。若以上两种都不行，请选择自定义，然后在服务器域名设定输入“cgi.arp2s2fnl2.com”
      • 若有音质方面的问题，请尝试随便调调DSP开头的那两个设置
    • 游戏设置的配置
      • 影响难度的设置
        • - 游戏难度建议选Hard或Lunatic。若选择自定义难度，可参照本教程第二部分 关于游戏难度 部分的推荐值设置
        - 将 生成点块最大时间更改为1000，可以阻止长条的生成，稍微降低难度，也可以根据自己的喜好调整
      • 影响观感的设置
        • - 更改 音符外观设定 ，选择一个你喜欢的音符样式
        - 关闭 是否连接所有并排的点块 与 是否连接上下相邻的滑块
        - 将 Perfect/Great/Good/Bad特效设定 后的特效改成自己喜欢的或者不晃眼的，并适当调节一下特效大小
        - 将 音符下落模式 改为None，让音符匀速下落，如果不喜欢音符匀速下落可以不改
        - 将 背景最低亮度 改为1，防止背景忽明忽暗地闪
      • 影响分数的设置
        • - 关闭 仅判定完美滑块 ，此项默认开启，导致滑块无论点还是滑都只加1分
    • 歌曲的单独配置
      • 打开歌曲的音轨界面，如果音轨较多，可以先查看简介，部分歌曲的简介中有推荐开启的音轨，照着调一般就能打。简介没有推荐配置的时，需要自己挑选适合的音轨
        • - 挑选音轨时，可以通过音轨左边写着的音色来判断该不该开，bass或者鼓类一般不建议开
        - 也可以通过静音和麦克风按钮来辅助自己手动筛选出那些比较主要的音轨
      • 若歌曲的音符下落速度过快或过慢，请调整游戏设置中的 音符下落速度 ，推荐值为0.8~2.3，日常建议1.5左右
  2. 进阶教程
    • 注册账号：请查看本教程第一部分，若注册失败或遇到问题，请查阅第五部分，或在群内求助
    • 文件导入：THMIX可以通过选择使用THMIX打开的方式来导入midi和soundfont文件，若想手动导入请看教程第四部分
    • 修改背景：见第四部分关于BackgroundImages文件夹的描述


第一部分 官网
-官网-

* 官网网址: https://thmix.org/ 或 https://asia.thmix.org/
* 使用中国移动的网络大概率访问不了网站，有条件的可以换个网络
* 访问官网时网址前面必须是https，使用http会被拒绝访问


  • home
    • 官网的首页。在此页面可以查看已注册玩家总数和当前在线人数，页面中间是各个下载渠道的链接和加入Discord群的链接
    • 页面的左上角有个菜单按钮，菜单点开后，可以前往其它页面，部分页面需登录后方可查看

    ◈ 关于菜单中的按键

    * 在菜单的中下方(PC端是在语言选择栏左边)，有一个刷新按钮，点击后能够重新加载网页，和点浏览器的刷新按钮没什么区别
    * 在刷新按钮的下方(PC端是右方)，有一个语言选择栏(默认语言显示的是English)，用于更改网站的显示语言
    * 在语言选择栏的下方(PC端为右边)是登录和注册按钮
    * 登录后，在语言选择栏的下方，登录和注册按钮变为一个可点击的列表按钮和当前用户的头像，点击头像可进入个人资料页面；点击形如 +▼ 的图标展开小菜单，在小菜单中，可以进行上传Midi等操作


  • midis
    • Midi商店。在这个页面，可以看到游戏内所有的可下载的Midi
    • 使用页面上方的搜索栏可以搜索Midi，此搜索栏特性与游戏中Midi商店的搜索栏特性相同
    • 除直接搜索外，也可以使用搜索框下方的筛选按钮选择特定的状态或特定专辑的Midi
    • 搜索完成后，还可以使用搜索区域下方的排序按钮，用其提供的方式对搜索结果进行重新排序
    • 页面下方是商店中的所有可供下载的Midi，Midi窗口中展示了Midi的部分信息
    • Midi窗口的背景是该曲的专辑封面
    • 窗口左下方是Midi的名字
    • 右上角是曲目的游玩次数，下载数，点赞数和喜欢数
    • 在封面图下方的信息栏中，自上而下分别显示着上传者，歌曲所属专辑，歌曲原名
    • 再下面是所有玩家的平均分数，平均连击数和平均准确度
    • 点击Midi的名字即可进入Midi的信息页面
    ◈ 关于Midi信息页面
    • Midi信息页面最上方四个图标从左到右分别显示的是游玩次数、下载次数、点赞次数和喜欢次数
    • 在四个图标下方的是Midi的名字和其作者，点击作者名字可以跳转到原出处查看此Midi(仅部分Midi支持该操作)
    • 再往下的是Midi上传者及上传时间，右边的图标显示的是Midi状态(Midi状态的说明见第二部分关于Midi信息界面的介绍)；Midi状态右边的是编辑按钮，只有在登录状态进入自己有权限编辑的Midi的信息页面时才有此按钮(正常情况下)
    • Midi状态下方的黑底方框显示的是玩家们的平均分数，平均连击数和平均准确度
    • User Rating下显示的是点赞与踩的数量的百分比(即好评度和差评度)
    • Description下的是简介内容，Source Artist下是原曲的作者；Source Album下是原曲的专辑；Source Song下是原曲的名字
    • 简介右侧的白底方框中显示了歌曲的通过状态，如Pass Rate下的数值是歌曲的通过率；Grade Status下是获得各个评分等级的比率
    • 再往下显示的是该曲排行榜的第一名的各项具体数值和排行榜
    • 在Midi信息页面的底部是评论区，登陆并通过人机验证后可以在此处发表评论
  • songs
    • - 这个页面目前没有东西，但是有一个叫customized的子页面，用于批量修改歌名方面的翻译，修改内容需要权限

  • translations
    • - 这个页面用于修改游戏内容的翻译，只能看看，修改内容需要权限

  • versions
    • - 这个页面是用于下载不同版本的TouhouMix，同时也能看到各个版本更新内容的介绍(然而官网的版本已经过时了)

  • users
    • - 这个页面摆放的是游戏的总排行榜。在这个页面中，可以看到各个玩家的排名(第一名的名次是#0)、游戏时间、游玩数量、总得分、平均连击数、平均准确度、演奏点数(即performance point，同时排行榜也是根据这项数值对玩家进行排名的)、取得S、A、B评分的数量
    - 在排行榜界面，可以通过点击玩家名字进入对应的玩家信息页面，在其它界面点击玩家名字也可进入(除了游戏内)，你可以看到这个玩家的排名、各个评分的数量、最近游玩的歌曲等信息
    - 单击在排行榜顶部的”Performance Ranking”下的年份可更换排行榜

    ◈ 关于个人资料页面
    • 在排行榜、菜单等页面中点击玩家头像进入对应的个人资料页面
    • 当登录自己的账号时，在Player Info的右边有两个按钮，第一个铅笔图标的按钮点击后进入个人信息编辑页面，玩家可以在其中修改名字、头像、个人简介和密码；第二个橡皮擦按钮点击后可以清除登录状态，点击后再刷新网页即可退出登录
    • 在玩家头像右边从上到下四项显示的是玩家的名字、账号注册时间、上一次登录时间(登录官网也算)和总在线时间
    • 右边四项显示的是玩家的总游玩次数、总得分、平均连击数和平均准确度
    • 再往下的的几个从左往右显示的分别是总游玩时间、总pp值、S到F各个评级的数量，右下是该用户在总排行榜中的排名，旁边的图表是排名变化趋势
    • “me!”栏显示的是玩家的个人简介
    • “Ranks”栏显示的是玩家的最好成绩，但一般显示不准
    • “Historical”栏记录着玩家每日的游玩情况，但是是以k线图的方式呈现
    • “Most Played Midis”栏显示着该玩家玩得比较多的Midi
    • “Recent Plays”栏显示着玩家最近玩的Midi


  • help
    • - 这个页面说是帮助页面不如说是反馈页面，此页面只有写着遇到各种问题时的反馈渠道

  • cards

    • - 此页面目前已被隐藏。需要登录账号后手动输入网址方可访问该页面。这个页面里有许多曾经计划用于做抽卡系统的卡牌图片，但是因为抽卡系统迟迟没有动工，所以这个页面暂时没有实际用途

  • soundfonts
    • - 此页面目前已被隐藏，不可通过菜单直接访问。目前没有任何作用。玩家上传的sf2文件会存在这里(然而就算不是sf2文件也能传)


-注册-
注册流程：
  1. 进入官网(https://thmix.org/)
  2. 点击左上角的菜单
  3. 点击Register(注册)
  4. 按要求输入用户名、邮箱地址和密码
  5. 输入邮箱内收到的验证码(输入时请确保网站连接不会中断；若收不到验证码，可尝试换个邮箱)

-Midi上传-
上传流程：
  1. 进入THMIX官网并登录上自己的账号
  2. 点击网页左上角的菜单按钮
  3. 点击位于头像上的 +▼ 图标，选择上传Midi
  4. 点击选择文件，选择要上传的Midi(选择完文件后稍等一会，上传完毕后会自动进入编辑页面)
  5. 编辑完毕后点击上传(upload)
  • 如果想在Midi上传完后编辑Midi信息，仅需在账号登录后进入Midi信息页，点击midi状态(一般显示为“PENDING”)旁的编辑按钮即可；若无编辑按钮请在当前网址的末尾加上 "/edit" 并直接访问
  • 若上传Midi时其自动跳转至已存在的Midi的信息页面，则说明要上传的Midi与其重复
  • Midi的ID同时也是显示在网址里的， 网址中Midis/后的一大串字符串就是Midi的ID，例如: "https://asia.thmix.org/Midis/123456/edit"的123456即为Midi的ID(同时也是Midi文件的Md5值)，当想要删除已上传的Midi或者上传一个已有Midi的新版本时，就会需要用上它了

编辑页面内容：
  • Cover：在该词的下方有一个选择上传文件的按钮，点击此按钮上传作为Midi的封面的图片，图片大小不可超过1MB，图片上传后会被裁剪成长条形
  • Meta：
    • derived from Midi id：如果上传的Midi是已有的Midi的新版本时，需要在此栏输入要替换的旧Midi的ID，同时下一项也需要填写旧Midi的ID，正常情况下无需填写
    • supersede Midi id (will kill old Midi)：在此项输入Midi的ID，即可删除对应的Midi，也可以输入当前Midi的ID来删除它自己，正常情况下无需填写；当Midi因为文件有问题等原因不能玩或不适合玩时，需要填写此项，填写并确定后无法撤销该删除操作
    • Midi name：Midi的名字
    • Midi description：Midi的简介。
      * 对于多音轨的Midi，可以通过写"开启/启用 + 音轨/Track + 数字" 来控制默认启用的音轨，如使用"开启音轨：00, 01"来默认开启0和1号音轨，数字间的分隔符只要不是数字就能识别(如""开启Track0和1)，在启用音轨时，默认之后读取第一个配置
      * 建议在写简介时写上标签形如"#东方project"之类的标签来方便玩家搜索到你的Midi，并且建议把歌曲的中日英翻译都单独作为一个标签
    • Midi artist name：Midi的作者。如果不清楚作者，推荐填Unknow
    • Midi artist url：输入Midi的原出处的链接，没有链接或是自己做的可不填
  • Source：
    • source album name：歌曲专辑名，如果是东方正作及小数点作的曲子，或者你有提前创建专辑，请在列表中选择，没有的话请自己输入或放着等管理来改
    • source song name：原歌曲名，请准确地输入此项，此项最好要填，不填会导致Midi下载后会和其它的没有所属歌曲的Midi扔一起
    • source artist name：原歌曲作者，请输入歌曲的歌手的名字，若不清楚原唱是哪位，可不填

        • Midi基本信息框的下方是此Midi的排行榜，排行榜根据玩家的pp值来进行排序，第一位排名是#0
        • 查看排行榜需要登录账号且账号总pp值达到1000及以上，否则排行榜上的玩家会被匿名处理，即无法看见玩家的昵称及头像
        • 在玩家名字下方显示的彩色数字是该玩家最高pp值的一次成绩的perfect、great、good、bad、miss数量
        • 右侧显示的内容从上到下，从左往右分别为:
          • pp值
            • pp值(performance point)的计算公式为: ln(1+「总分数」) × 「准确度」 2
          • 完成度
            • 完成度为 实际pp值/理论最大pp值。个位数字若为0则不显示个位上的数字
          • 评分等级
            • 评分等级有Ω、S、A、B、C、D、E、F等，除Ω和F外，其他等级均可再分出三档，如A+和A-，但S是另外分为SSS和SS，评分等级仅根据准确度给出，与得分或pp值无关，不同评分等级对应的结算界面左侧显示的立绘角色及表情不同，同时该人物立绘会在返回专辑界面时显示在右侧(即原本显示琪露诺的位置)
          • 小太阳图标后的数字为得分
          • 锁链图标后的为最大连击数
          • 准星图标后的为准确度
        ◈ 关于下落的音符
        • "Note"(或称为"音符"等)一般同时指点键、滑键和长条三种音符
        • "点键"(或者称为"点块"等)指的是下落到判定线时需要单次点击的音符
        • "滑键"(或者称为"滑块"等)指的是在落到判定线时需要手指点击或者提前放在判定线上才能消除的音符，若滑键下落至判定线前手指就已放在判定线上，则只加1分；若打开了 【仅判定完美滑块】，则无论滑动还是点击都只加1分
        • "长条"指的是落到判定线上需要玩家长按并在其结束时准时松手的音符，在按住长条时可以左右移动手指来移动长条，若长条结束时没有准时松手，则尾判处只得一分
        • 教程中出现的所有"音符"均指的是下落的点键、滑键与长条
        • 单次点击(所有类型的音符)的分数的计算公式为:
          单击分数 = 100 ×「连击加成」×「准确度权重」
          ╰连击加成 = 0.0809 + 0.199 × ln(「连击数」)
          ╰准确度权重 = 1.06 × e (-7.17 ×「Note偏移时间」)
        • 单个Perfect判定的准确度为1；Great判定为2/3；Good为1/3；Bad与Miss为0

-总设置-
总设置位于游戏内，在专辑界面和歌曲信息界面点击屏幕右上角的"⚙"️(齿轮图标)即可进入，将其称为总设置主要是与游戏设置做区分，同时也是因为它位于外部。以下各项后的中括号内的内容分别是该项的中文翻译
  1. Online Mode [线上模式]
    2. * 若关闭此项，则游戏内需要联网的内容都会隐藏，包括：Midi信息界面的排行榜和评论区、商店按钮以及游戏顶栏右部的账号登录功能
    * 打开后需要重启游戏才能连上服务器
  2. Display Language [语言选择]
    3. * 更改游戏语言
  3. Translate User Generated Content [全局內容翻译]
    4. * 设置是否翻译歌曲名字、专辑名字及玩家评论等内容，如果嫌歌曲、专辑名称机翻过多，可关闭此项，设置翻译不受影响
  4. Network Endpoint [服务器网络节点]
    5. * 选择游戏连接的服务器，如发现无法连接至游戏服务器时，可尝试更改此设置后重启游戏
  5. Custom Server Domain Name [服务器域名设定]
    6. * 设定游戏连接的服务器的域名，当官网没了或者换网址了或者官网连不上时可能用得上，正常不需要改
    * 当连接不上服务器时可以试试填入这个: "cgi.arp2s2fnl2.com"
  6. Gameplay Result Format [成绩展示重点]
    7. * 设定游戏的歌曲排行榜在显示成绩时着重展示的内容(即字号最大的那个东西)，可选择着重显示 分数 或 完成度(实际pp值除以理论最大pp值)
  7. Audio Reverb Preset [音频混响设定]
    8. * 更改混响的音效，应该类似手机自带的均衡器吧，具体效果自行尝试
  8. Synthesizer Sf2 File Path [音源文件路径]
    9. * 将.sf2文件移动至「游戏文件目录」中的 Sf2 文件夹后打开游戏即可选择导入的音源文件，且无需重启游戏即可生效
    * 导入音源文件也可以直接选择用THMIX打开音源文件，游戏会自行导入
  9. Synthesizer Volume [合成器音量]
    10. * 更改游戏音量，单位为分贝，数值越大音量越高，但是音质会变差，所以不建议调过高，适当即可
  10. Synthesizer Low Pass Filter Parameter [合成器低通滤波器参数]
    11. * 设置为0相当于关闭此功能，当设置的数值越高时，游戏对电音吉他等较为尖锐的声音的抑制就会增强，具体多少合适还请自己调试
  11. Synthesizer Render Interval [合成器渲染间隔]
    12. * 调成1则为最佳的声音效果，数值高于1效果和1相同，低于1时，虽然对设备性能的需求会些许降低，但是听着不是很好
  12. Synthesizer Max Voice Count [最大声音數量]
    13. * 设置同时发声的最大音符数量，将此项数值调小可能能减轻游戏在播放较为复杂的Midi时的压力，一般无需修改
  13. DSP Sample Rate Down Scale [DSP采样率缩减]
    14. * 如果游戏玩着会卡顿、掉帧或出现音质方面的问题，可以把该项调小，但音频延迟也会有所增大。如果手机好建议还是用较高的采样率以获得更好的音质
  14. DSP Audio Buffer Up Scale [DSP音频缓冲放大]
    15. * 如果游戏玩着会卡顿或掉帧可以把此项的数值调大，能够减轻卡顿等症状，但会增加音频延迟
  15. Lane Count Theoretical Audio Delay [理论音频延迟]
    16. * 这里显示的数值就是理论上点击音符到发出声音的延迟，一般建议调整第10项和第11项来获得一个最低的音频延迟，因为音频延迟只能在这里调整
  16. MP3 Audio Desync Threshold [MP3音频非同步阈值]
    17. * 当歌曲有对应的音频可播放时，游戏会根据绝对时间去同步midi和音频的播放，当两者时间差超过此阈值时会重新同步
  17. Reload Resource Bundle [重载内置资源]
    18. * 让游戏重新加载内置的资源包，包括翻译文件、封面图、Midi及相关文件，当误删文件导致进入游戏一片空白时可用此项来重载内置资源，该操作不会删除已下载的Midi
  18. Reload Translation Cache [清除翻译文件]
    19. * 删除翻译文件，让游戏重新加载翻译


-游戏设置-
  • 点击位于歌曲开始播放时屏幕正中间的齿轮图标的按钮进入游戏设置
  • 在打歌过程中点击暂停，再点击三个按钮下方的设置按钮也可进入游戏设置
  • 称之为游戏设置主要是与总设置做区分，同时方便介绍

第I栏 播放设置
  1. Touch Judging Criteria [判定方式]
    • Nearest [最临近]
      • 使音符的判定范围变为半径为【音符判定宽度】所设定的数值的圆，手指在以外区域的点击不会生效
    • Best Timing [最佳时机]
      • 使游戏优先判定时机最佳的音符，判定范围为垂直方向的全屏判定，判定范围的宽度为【音符判定宽度】所设定的数值
    • Nearest from Judge Line [距判定线最近]
      • 使游戏优先判定离判定线最近的音符，判定范围为垂直方向的全屏判定，判定范围的宽度为【音符判定宽度】所设定的数值
    • Best Timing (First In, First Out) [最佳时机(先现先判)]
      • 使先进入屏幕范围的音符会先被判定，判定范围为垂直方向的全屏判定，判定范围的宽度为【音符判定宽度】所设定的数值
    • 上面几项有几个看着好像没什么区别，实际可能在某些情况下会有所不同，若遇到判定问题可以尝试更改判定方式
  2. Check Instant Blocks When Finger Up [抬手时判定滑块]
    3. * 若启用该项，在抬起手指时，处于判定范围内的滑块会进行判定
  3. Only Perfect Instant Blocks Before Judge [仅判定完美滑块]
    4. * 若打开此项，在滑块落至完美判定区间的范围前无法被判定，即使主动点击，但当滑块经过判定线后，仍能出现Great或Bad等判定
    * 打开此项后，点击或划滑块仅能获得1分，如果要冲高分，不建议开启
  4. Performing Mode [弹奏模式]
    5. * 选择no后，即使不去点击下落的音符，选择了的音轨也会正常发声
    * 一般建议打开，能够增强体验，关了之后相当于没有Key音了，玩着可能没有自己在打的感觉
    * 若是经常miss，导致不能好好听歌还是建议选择关闭此项
  5. Use System Stopwatch [使用系统时钟]
    6. * 通过使用系统时钟(硬件时钟)来提高精度，如果游戏内音频时而提前时而延后可以启用
  6. Play Ready Animation [播放准备动画]
    7. * 关闭后，音乐开始前不会有准备动画，不建议关闭
  7. Vibrate on Touch [触摸震动反馈]
    8. * 选择是否在点击音符时使设备震动
  8. Vibration Duration [震动持续时间]
    9. * 设置点击音符时震动的持续时间，单位：毫秒

第II栏 颜色设置
  1. Randomize Block Color [随机音符顏色]
    2. * 随机设置下落音符的颜色。由于光柱特效的颜色与音符的颜色相同，所以当玩家使用自定义皮肤时该设置项相当于随机设置光柱特效的颜色
  2. Sync Color Across Block Type [统一音符颜色]
    3. * 使点块、滑块和长条的颜色统一，同时也会统一光柱特效的颜色，但打开此项后音符颜色无法随机
  3. Instant Block Color [更改滑块颜色]
    4. * 通过输入RGB颜色编码或颜色的英文名字来改变 滑块 及其光柱特效颜色，仅支持部分内置皮肤
  4. Short Block Color [更改点块颜色]
    5. * 通过输入RGB颜色编码或颜色的英文名字来改变 点块 及其光柱特效颜色，仅支持部分内置皮肤
  5. Long Block Color [更改长条颜色]
    6. * 通过输入RGB颜色编码或颜色的英文名字来改变 长条 及其光柱特效颜色，仅支持部分内置皮肤

第III栏 皮肤设置
  1. Custom Block Skin Texture Filter Mode [皮肤纹理过滤模式]
    2. * 更改对音符皮肤的渲染方式：
      - 选择Nearest会使音符皮肤的各个像素之间的分界很鲜明(不同颜色之间)，使音符皮肤看着棱角分明，一股像素风
      - 选择Bilinear会使不同颜色的像素点间有一个模糊过渡，使音符皮肤看着更圆滑，或者说模糊
  2. Block Skin Preset [音符外观设定]
    3. * 直接选择已导入的或内置的音符皮肤的名称即可更换皮肤
    * 若正在使用的音符皮肤被删除，会导致此项无法更改，解决措施请见 第五部分 常见问题

第IV栏 背景设置
  1. Custom Background Image Size Mode [背景尺寸模式]
    2. * 更改背景图片适配屏幕的方式：
      - Cover为覆盖全屏幕，当图片尺寸不足以直接覆盖屏幕时，图片会被放大，直到能够完全覆盖屏幕
      - Fit为适应屏幕大小，图片会完整地显示在屏幕上而不会被缩放
  2. Background Image Preset [背景图像设定]
    3. * 设置打歌过程中的背景图像，图片需要放置在「游戏文件目录」下的BackgroundImages文件夹才能被使用
  3. Background Image Color [背景图像颜色]
    4. * 设置叠在背景上的纯色图层的颜色和透明度，如“#AABBCCDD”中，AABBCC为RGB颜色编码，DD为透明度，数值越大越透明，最大为FF(十六进制数字，既255)
  4. Background Image Flash Color [背景闪光颜色]
    5. * 修改打击音符时背景发光的颜色，调整方式同背景图像颜色
    * 若要关闭，调为与背景图像颜色相同的参数即可
  5. Background Dim Alpha [背景最低亮度]
    6. * 背景的不透明度会随着准度的变化而变化，准度越高不透明度越高，也就是看着越亮
    * 当此项设置为1时，即便准度为0，背景也不会变暗；若设置为0，当准度为0时，背景会完全变黑

第V栏 显示与判定
  1. Lane Count [下落轨道数量]
    2. * 在默认的12轨下，游戏在生成音符时，每个音都会从右往左依次对应一个轨道，因此隔了一个八度(12个音)的两个音会生成在同一个轨道。当轨道数过多时，有的轨道可能会没有音符
    * 设置音符的下落轨道数，数值过高可能会提升出现重叠的音符的概率
    * 轨道数更改后，生成的铺面可能会和之前的看起来完全不同，所以此设置项在某些时候能够影响读谱难度
  2. Block Size [音符尺寸大小]
    3. * 调整音符的显示宽度，音符判定区域大小不受其影响
  3. Block Judging Width [音符判定宽度]
    4. * 调整音符的判定范围的宽度，不影响判定范围的高度
    * 但在扫描线模式中，该项影响的是音符判定范围的半径
    * 不建议调太大，容易误触
  4. Min Block Spacing [最小横向音符间距]
    5. * 控制音符间的水平最小距离，数值太大会导致没有音符下落，太小可能会有部分音符过于接近屏幕边缘，所以在一定程度上影响了游戏难度
  5. Canvas Horizontal Padding [安全区域宽度设定]
    6. * 控制生成的音符与屏幕左右其中一边的最小距离，会影响音符生成位置，变相影响难度

第VI栏 判定线设置
  1. Judge Line Position [判定线高度]
    2. * 调整判定线距离屏幕底部的距离
    * 在扫描线模式中，该项设置的是扫描线上下运动时距离上下屏幕边界的最近距离
  2. Judge Line Thickness [判定线粗细]
    3. * 调整判定线厚度，不影响判定，只影响外观

第VII栏 下落设置
  1. Playback Speed Scaling [播放速度设定]
    2. * 调整音乐的播放倍率，数值越大速度越快，速度低于1时，游戏分数将不会上传
  2. Cache Time [音符下落速度]
    3. * 调整音符在判定线以上范围的下落速度，数值越小速度越快，默认为2(推荐调整范围:0.7~2.5)
    * 在扫描线模式中，该项为设置扫描线的运动速度(推荐调整范围:3或更高)
  3. Cache Easing Type [音符下落模式]
    4. * 设置音符在判定线以上范围的下落方式：
    下落方式选项卡
    左选框 翻译 介绍
    None None 无任何下落效果(匀速下落)
    Linear 线性下落 匀速下落(和None效果相同)
    Quad 正常的减速下落 与Sine相同，但初始速度更快，减速过程更迅速
    Cubic 稍快的减速下落 与Quad相同，但初始速度更快，减速过程更迅速
    Quart 较快的减速下落 与Cubic相同，但初始速度更快，减速过程更迅速
    Quint 极快的减速下落 与Quart相同，但初始速度更快，减速过程更迅速
    Sine 平缓的减速下落 从出现时开始，速度逐渐减慢，到达判定线时减到最慢速度
    Expo 疯狂的减速下落 与Quint相同，但初始速度更快，减速过程更迅速
    Circ 特殊的减速下落 先以较快的速度运动，距离判定线还有一段距离时剧烈减速，然后以十分缓慢的速度下落至判定线
    Back 缓和的过线折返 音符落到判定线下方一段距离后折返至判定线位置再继续下落，折返时的速度较慢。视觉效果类似于物体以较慢的速度落入水中，在水面(判定线)上下浮动
    Elastic 猛烈的过线折返 以十分快的速度落下，到达判定线下方的更远距离后弹回，随后速度迅速衰减，在判定线上下往返几次后落下。视觉效果类似于速度较快的物体落入水中，然后被弹出水面(判定线)后又重新落入的水中并浮动几下
    Bounce 触底反弹 音符在到达判定线后会被弹回数次后再下落。视觉效果类似于乒乓球在地面(判定线)上弹
    右选框 翻译 介绍
    None None 不改变左边选择下落方式所产生的下落效果
    In 进入 默认设置，会使左边选择的下落方式与其描述相同(同None)
    Out 退出 使左边选择的下落方式产生效果的时机与描述相反。如：减速下落变为出现时以最慢速度逐渐加速至最快速度；触底发弹变为在出现时就弹跳
    InOut 进出 使左边选择的下落方式同时产生In和Out的效果。如：减速下落变为以最慢速度出现并加速，路程过半后又开始减速，到达判定线时回到最低速度；触底反弹变为在出现时和落到判定线时都会反弹数次

  4. Grace Time [音符过判定线后的下落速度]
    5. * 同本栏第2项，但调整的是音符过判定线后的下落速度
  5. Grace Easing Type [音符过判定线后的下落模式]
    6. * 同本栏第3项，但影响的是音符过判定线后的下落方式，此时应将判定线位置理解为出现位置，屏幕底部理解为判定线位置
    * 此数值会影响音符Miss的时机，因为音符离开屏幕后才会被视为Miss
    * 在歌曲播放完毕后，游戏会等待一段足以让所有将要Miss的音符落出屏幕的时间，即使没有音符将要Miss

第VIII栏 生成设置
  1. Instant Block Max Time [生成滑块最大时间]
    2. * 控制滑块的生成时间，使长度低于此时间的Note变成滑块，调为0则不会生成滑块，但受其他设置影响仍可能会有滑块生成
  2. Short Block Max Time [生成短块最大时间]
    3. * 使低于此时间，高于滑块生成时间的音符变为点块，高于此时间的将变为长条
  3. Max Simultaneous Blocks [最大并排下落音符]
    4. * 控制同时下落的音符的最大数量，使并排下落的音符的数量不会超过此值，但仍可能遇见多个音符同时下落，一般是音符没有完全对齐导致的
  4. Min Tap Interval [连点最小时间间隔]
    5. * 使一个点块后跟着的与其间隔时间低于此时间的所有点块变成滑块，对变成滑块的点块不适用，且它们需在同一下落轨道
  5. Min Touch Cooldown Time [最小点击冷却时间]
    6. * 最小点击冷却时间，在一段纵连之中，如果上下相邻的音符之间的间隔时间超过最小点击冷却时间，则后面的音符变为滑块
  6. Max Touch Move Speed [手指最大移动速度]
    7. * 使各个相邻的音符(无需同一轨道)之间的水平距离在设置的数值所推算出来的距离之间。例如设置成100，然后两个离得近的音符之间的时间差是0.1s，那么它们的水平距离不会超过10像素(0.1s ×
    100pixels/s)(有待证实)
    * 该项也对一连串的下落音符有效，能够有效减小满屏下落的音符的接取难度
  7. Max Block Coalesce Time [每个批次最大时间]
    8. * 控制每组下落的音符的时间间隔，因为每组下落的音符数量是固定的(应该)，所以当每个批次之间的时间间隔越大，音符就会越稀疏，导致难度降低，建议调整范围为0~1

第IX栏 连线设置
  1. Generate Short Connect [是否连接所有并排的点块]
    2. * 选择是否生成连接线，连接并排下落的点块、滑块和长条
  2. Generate Instant Connect [是否连接上下相邻的滑块]
    3. * 选择是否生成连接线，连接同一下落轨道内的时间差在设定范围内的数的音符
  3. Instant Connect Max Time [被连接音符的最大时间差]
    4. * 设置被连接音符之间的最大时间间隔，低于此数值的同音轨音符就会被连接，前提是有打开【是否连接上下相邻的滑块】

第X栏 判定与延迟
  1. Judge Time Offset [按键判定延迟偏移]
    2. * 调整判定提前或延后的时间，不会使音符提前或延后生成，所以解决不了音频延迟的问题
  2. Perfect Time [Perfect判定范围]
    3. * 控制perfect评分的判定时间范围，此数值无法更改
  3. Great Time [Great判定范围]
    4. * 控制great评分的判定时间范围，此数值无法更改
  4. Good Time [Good判定范围]
    5. * 控制good评分的判定时间范围，此数值无法更改
  5. Bad Time [Bad判定范围]
    6. * 控制bad评分的判定时间范围，此数值无法更改

第XI栏 光效设置
  1. Flash Alpha [光柱不透明度]
    2. * 修改光柱的不透明度，有效数值为0~1，修改为0即可完全隐藏光柱特效
  2. Perfect Spark Preset [Perfect特效设定]
    3. * 点击选框或左右两旁的按钮更改按键特效的样式，更改时屏幕中间会出现一个特效的预览，其大小由下一个设置项决定，目前有一百多个可选的特效，其中120个来自Unique Projectiles
  3. Perfect Spark Scaling [Perfect特效大小]
    4. * 调整按键特效的半径，调成0即可隐藏特效
  4. Great Spark Preset [Great特效设定]

  5. Great Spark Scaling [Great特效大小]
    6. * 调整按键特效的半径，调成0即可隐藏特效
  6. Good Spark Preset [Good特效设定]

  7. Good Spark Scaling [Good特效大小]

-Midi导入-
  • 方法1：找到要导入的Midi，手动选择打开方式，并使用THMIX打开
  • 方法2：找到要导入的Midi，将Midi复制或者剪切到「游戏文件目录」内，Midi会被放在名字为"Local Midis"(若有开启 【全局内容翻译】，则专辑名字为"本地Midis")的专辑内
◈关于Midi
  • Midi在THMIX中的作用相当于一个铺面文件加上音乐，因为THMIX是通过Midi来生成铺面的。也正因如此，如果游戏中没有心仪的曲子，可以很方便地通过在其他网站寻找Midi并导入到THMIX中来游玩或者是上传到THMIX官网来和大家一起玩
  • 游戏内置的Midi基本来自 東方ピアノEasyモード ，作者为DMBN
  • 截止至2025年1月18日，THMIX官网已收录了大约2159个Midi，内置Midi约占35.0%
  • 从441版开始，THMIX官网内所有INCLUDED状态的Midi都将会被打包进游戏作为内置Midi
  • 若希望自己的Midi能够成为游戏的内置Midi，请确保Midi文件为原创或已经获得作者授权，并告知开发者或Midi管理人员
  • 若想要制作Midi，请自行上网学习

-音源导入-
  • 在文件管理器中找到音源文件，然后选择用TouHouMix打开即可成功导入
  • 找到需要导入的音源文件并将其移动至「游戏文件目录」下的Sf2文件夹中下即可
  • 若要使用音源文件请在游戏中使用【音源文件路径】选择该音源文件
    • ◈关于音源
    • 游戏中使用的默认音源是GS SoundFont
    • 需要注意的是，由于音源文件可能使用着不同的标准，若midi需要的音色缺失，缺失需要的音色的音轨会从该音源文件的可用音色中按编号从小到大重新选择一个音色(也就是从大钢琴音色开始，直到可用)

-人声导入-
  • 游戏支持导入mp3, ogg, wav和flac格式的音频作为歌曲的人声部分
  • 首先将MP3文件命名为 「Midi文件名或其md5的值」.mp3，例如：“miko_easy.mp3” 或 “fc79454eb16f6a6ff1e5808592cfc58c.mp3” 都能作为游戏中内置歌曲 永远的巫女 的人声轨文件的名字，其中后者是midi文件的md5值
  • 将命名完毕的音频文件移至「游戏文件目录」下，随后重启游戏即可
  • 若游戏能够识别并将音频文件匹配至对应的歌曲，则歌曲的信息框上会显示mp3的长度和采样率。同时，在音轨界面还能够对播放的延迟和音频音量进行调节

第五部分 常见问题
〖观前提醒〗
☆ 各个问题的解决措施均按推荐顺序排列，请放心尝试
☆ 部分苹果手机做不到的操作请iOS用户自行跳过

  1. 官网无法访问
    2. * 请确保访问的是(https://thmix.org)，若是使用http则连接会被官网主动切断；同时，因为asia服务器暂时没了，所以(https://asia.thmix.org)理论上访问不了
    * 检查自己的网络，使用移动的网络或宽带有很大可能会无法访问官网，所以访问不了的时候可以试试换个网络
    * 可以尝试使用科学上网
    * 放弃治疗

  2. 收不到验证码或验证码错误
    3. * 若收不到验证码，可多尝试几次或者换个邮箱注册
    * 若输入验证码的时候提示验证码错误，请先确保输入无误；其次，注册时设备与官网的连接不能中断，请确保查看验证码的过程中，浏览器与官网的连接不会断开
    * 若自己的邮箱都收不到验证码，可以试试用网上的临时邮箱
    * 若问题还是无法解决，可以托别人代为注册

  3. 为什么官网不能下载Midi
    4. * 因为涉及版权等问题，所以官网不提供下载Midi的服务

  4. 怎么在上传完Midi后再次编辑
    5. * 请见关于Midi上传的介绍

  5. 怎么删除自己上传的Midi
    6. * 请见关于Midi上传的介绍

  6. 为什么没有登录按钮
    7. * 点开屏幕右上角设置按钮，将【线上模式】改为Yes，然后重启游戏即可

  7. 语言选择中文却没有中文
    8. * 重启一次游戏或返回上一界面再回来
    * 请先确认是否能连接上服务器，屏幕顶部的延迟要是显示出来了一般就代表已经成功连接上服务器，如果连接不上服务器，请看问题一
    * 如果不是网络问题的话呢，可以多等等或者暂时放弃治疗

  8. 游戏内无网络连接
    9. * 使用浏览器看看能否访问官网，若不能请看问题1，若问题解决后能访问官网，则说明不是当前网络的问题，无需使用科学上网工具
    * 看看进入游戏时延迟是否有数值，如果有说明可以连接至服务器，可以多等等
    * 更改【服务器网络节点】并重启游戏
    * 尝试使用数据流量或者梯子
    * 放弃治疗

  9. 翻译为什么这么渣
    10. * 如果指歌曲及专辑名字翻译太渣，可以先选择更换语言为繁体中文
    * 如果翻译错误还是很多，可以选择关闭【全局内容翻译】，眼不见为净
    * 如果是设置项的翻译错误，例如与此教程内的翻译不同，则是因为游戏调用了内置的离线翻译文件而不是在线获取新的翻译。可以在确保游戏能够连接上服务器的情况下，点击【清除翻译文件】并稍等片刻或重启游戏等待翻译重新加载

    11. * 如果反复尝试后均提示THMIX无响应，请尝试清除应用数据或更换旧版本后重试
    * 如果THMIX会因为一些特殊原因在启动时关闭(如因为没有开启网络而自动关闭)，请确认自己的安装渠道是否安全，可尝试去官网下载正版或更换旧版游玩的呢

    12. * 首先需要确保【线上模式】处于打开的状态

  14. 专辑界面空白或界面重叠
    15. * 重启游戏
    * 点击右上角设置内的清除资源包
    * 删除「游戏文件目录」下的JsonStorage文件，并重启游戏。该操作会导致游戏设置内容丢失，不过不会删除已下载的Midi
    * 在手机设置的应用管理中找到THMIX并删除应用数据，该操作会丢失所有游戏数据
    * 重装游戏，不过不如删除应用数据快和简单

  15. 无法在Midi商店内下载Midi
    16. * 若是点击后一直卡在0%，则可能是网络问题，可试试问题8的解决方案
    * 若是显示已下载，而专辑列表中没有，则需在「游戏文件目录」中的DB文件夹中的midis文件夹下删除对应文件
    * 如果都不行的话，可以试试在玩家交流群中下载Midi整合包

  16. 播放交响乐的观感很差
    17. * 如果是觉得音质很差，可将【合成器渲染间隔】调整为1
    * 交响乐观感差，极大概率是因为各个乐器的音量不够平衡而导致的，但游戏目前还无法针对特定的音色调节音量，所以目前只能采取以下两种替代方案：
    1. 更换一个音源文件。不同的音源文件，各个音色的音量也有所不同，要是有合适的音源的话，能很大程度上提升交响乐的观感
    2. 调整部分音轨的音色。可以将那些声音很大的音轨的音色换为声音较小的音色，但是用这种方法，想要把Midi调校到一个很好听的地步十分困难，一般可以直接把所有音轨都统一成一种音色，不过那样听的就不是交响乐了
  17. 游戏卡顿、掉帧或有杂音
    18. * 重启游戏
    * 尝试调小【DSP采样率缩减】和调大【DSP音频缓冲放大】
    * 调小【合成器音量】
    * 若有杂音或音质不佳，请将【合成器渲染间隔】调整为1，若游戏卡顿，可往小了调
    * 调小【最大声音数量】的数值，可能没用

  18. 游戏难度太低了
    19. * 进入歌曲信息界面后，直接在屏幕左上角的难度选框中选择除custom以外的难度，或选择custom并手动调整游戏设置-VIII-第3~7项的参数
    * 在音轨界面中多启用几条音轨
    * 适当调小【最小横向音符间距 & 安全区域宽度设定】的数值。这两项数值太大会顶掉音符
    * 将【生成短块最大时间】调为1，使长条正常生成，增加长条尾判的那个音符量
    * 适当调大【下落轨道数量】的数值。轨道数太小会吞掉少量音符
    * 若音符总量还是不够，我也没办法

  19. 设置改完后怎么会被还原
    20. * 游戏内的部分设置项会直接影响游戏难度，所以需要在custom难度下才能更改，请先将难度改为Custom
    * 可能是点到屏幕左上角的重置按钮了，返回按钮在左下角
    * 如果仍无法更改则说明该项不支持更改

  20. 已下载的Midi如何去删除
    21. * 在游戏中的歌曲信息界面，点击左下角音轨按钮右边的垃圾桶图标，然后点右边想要删除的midi小框左上角的叉叉按钮即可，再次点击垃圾桶图标隐藏叉叉按钮

  21. 滑键间的连接线能否关闭
    22. * 连接线纯属摆设，如果觉得碍眼，可以关掉【是否连接上下相邻的滑块】

  22. 音符间的距离太近或重叠
    23. * 将【最小横向音符间距】的数值稍微调大
    * 查看是否为轨道数量的问题，若轨道数量太大，相邻音符之间的间距会很小甚至重叠；可尝试调小轨道数量(例如调成12)
    * 如果不是轨道数量问题的话，可能是【手指最大移动速度】的设置问题，此设置项数值太小会使音符之间距离被拉近，可以尝试调大(建议500~4000)
    * 通过小幅降低游戏难度来使自己更容易应对:
    • 将【连点最小时间间隔】数值略微调大，比如0.1，使两个重叠的点键变为一个点键与一个滑键
    • 将【最大并排下落音符】数值调小，例如2，使重叠的并排点键更容易对付
    • 将【生成短块最大时间】的数值调大，例如1000，使长条不生成
    • 适当调大【手指最大移动速度】的数值

  23. 按键特效晃眼，能否更换
    24. * 如果说的是那个爆炸特效的话，通过【Perfect/Great/Good/Bad特效设定】的选框或其两旁的左右小箭头按钮更换按键特效
    * 如果想要2.x版本的那个特效，那么很抱歉，那个特效已经没了
    * 如果没有心仪的特效并想关闭特效的话，通过将【Perfect/Great/Good/Bad特效大小】后面的数字改为0就能隐藏爆炸特效了
    * 如果说的是那个光柱特效，那么可将【光柱不透明度】调为0

  24. 皮肤无法显示或进行更换
    25. * 请确认皮肤是否本身就存在问题，例如缺少必要的贴图
    * 若将【音符外观设定】更改后退出再回到设置后显示的内容发生变动，则可能是因为上一个选择的皮肤被删除而触发了bug。可以在「游戏文件目录」中的Skins文件夹下创建一个名字和之前被删除的皮肤名字一样的文件夹(或将皮肤文件移回来)，然后再回到游戏设置中，就能正常更改皮肤了
    * 点击游戏设置左上角的重置按钮

  25. 歌曲封面为什么是模糊的
    26. * 因为开发者比较怂，怕涉及版权问题
    * 当登录的账号的总pp值达到1000时就能够正常显示这些图片了。所以，努力打歌吧
    * 若总pp超过1000却仍显示模糊图片，则需手动清理「游戏文件目录」中的WebCache文件夹下的模糊图片。注意，请勿直接删除WebCache文件夹，这会导致歌曲文件丢失
    * 如果图片重新加载后仍为模糊图片，请催kai快点修bug
  26. 音符轨道的排列过于分散
    27. * 若开启了键盘模式，但是只设置了四个按键，导致轨道排列稀疏，可以调大【安全区域宽度设定】，加大两侧留白区域，使轨道在中间紧凑排列
    * 同上，或者修改跟 显示与判定 有关的几个设置项
